Mark's Code Fractal: A Concept Reborn

During the intervening years between when I first worked on MSS Code Factory with JDK 1.6 and today, a lot of technology has evolved, improved, and simplified the art of programming. What it hasn't done is eliminate the need for painstaking attention to all the details in making modern infrastructure frameworks "go." LLMs while promising at first have proven vulnerable to their own unique classes of errors, going far beyond simple hallucination of magic packages that solve the problem to repeating bad examples of code that were popular at the time they scraped their core information. The end result is that you can't trust LLM-authored code any more than you can that produced by a smoking-fingers-on-the-keyboard team of junior programmers hired through some contractor: mistakes will happen, and they'll be different almost every time they do.

What Mark's Code Fractal will be bringing to the table is consistent output code and mapping of the Business Application Models to application code and data files. It isn't intended to be a "4GL" or an "application generator", but a tool for managing problems with the foundational code of an application by letting you update and modify the rules that define how to produce the code whenever you find a problem with an implementation, and thus achieve real ISO9000-series feedback correction of not only your current application, but any and all applications produced by the corrected rule base.

Although I've focused on Java for the past while, there are examples of C++ rules in cfkbase-v2_13, there used to be samples of C# rules, and there are a plethora of examples of JDBC stored procedure and binding layer implementations for most of the major databases in the old rule set. But all of that is only gleaned for useful fragments at this point; the modern magic happens elsewhere: in org.msscf.msscf.v2_13.cfkbase/src/main/resources. There you'll find the server.markhome.* rules that define the 3.1 code base along with the foundational class implementations and interfaces for cflib in the 3.1 core. Only the Java core code is being uploaded at this point in time - as it is there are now over 50 repos under my id...

I'm guessing I've got about 9-15 months of work left to achieve my 3.1 vision. But it's far enough along to share with the world...
